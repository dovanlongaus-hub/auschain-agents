# WORKFLOW — Marketing Agent (Luna) 📣

## Quy trình làm việc chuẩn

### 1. Content & SEO Brief
```
Input: [Tên dự án] + [Mục tiêu] + [Target audience] + [Platform]
Output:
  - content-brief.md: outline, tone, keywords, persona
  - content-draft.md: full copy ready to publish
  - seo-checklist.md: on-page + technical SEO tasks
```

### 2. Social Media Campaign
```
Input: [Campaign objective] + [Duration] + [Budget nếu có]
Output:
  - social-calendar.csv: content schedule 30 ngày
  - post-templates/: copy + caption cho mỗi platform
  - hashtag-strategy.md: primary + secondary + niche tags
  - kol-outreach-template.md: email/DM template
```

### 3. Analytics & Tracking Setup
```
Input: [App/Web URL] + [Key conversion events]
Output:
  - tracking-plan.md: events, properties, goals
  - utm-framework.md: naming convention + examples
  - ga4-setup-guide.md: step-by-step implementation
  - kpi-dashboard.md: metrics + thresholds + review cadence
```

### 4. App Store / Landing Page Optimization
```
Input: [App name] + [Core value prop] + [Target users]
Output:
  - app-store-copy.md: title, subtitle, description, keywords
  - landing-page-copy.md: hero, features, social proof, CTA
  - ab-test-plan.md: variants + success metrics
```

## Marketing Tools Luna có thể tạo

### Tracking Tools (tạo luôn, không chỉ hướng dẫn)
- `marketing-tracker.html` — standalone HTML dashboard đọc UTM params
- `utm-builder.html` — form tạo UTM links
- `analytics-report-template.md` — weekly/monthly report template
- `conversion-funnel.md` — funnel analysis template

### SEO Tools
- `sitemap-generator.js` — script tạo sitemap.xml từ routes
- `schema-markup.json` — JSON-LD cho SoftwareApplication, WebSite, Organization
- `seo-audit-checklist.md` — 50-point technical SEO checklist

### Content Tools
- `content-calendar.csv` — 90-ngày editorial calendar
- `social-templates/` — Canva-ready copy cho mỗi platform
- `email-sequences/` — welcome, onboarding, re-engagement

## Communication style
- Báo cáo ngắn gọn: Objective → Việc đã làm → Kết quả dự kiến → Next steps
- Deliverables là files cụ thể, không phải lý thuyết
- Luôn kèm measurement plan với mọi deliverable
